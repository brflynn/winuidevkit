# Rust Language Pack

**WinUI3 for Rust via [windows-rs](https://github.com/microsoft/windows-rs) — no new bridge needed.**

## Quick Start

```powershell
# 1. Install (downloads from WinUIDevKit release — no clone needed)
irm https://github.com/brflynn/WinUIDevKit/releases/latest/download/install.ps1 -OutFile install.ps1
.\install.ps1 -Language rust

# 2. One-time: install Windows App SDK
winuidev setup

# 3. Create and run a project
winuidev init MyApp
cd MyApp
winuidev run
```

## How It Works

The `windows` crate (Microsoft 1st-party) generates Rust bindings directly from
`.winmd` metadata files — no .NET runtime needed. Rust calls WinRT interfaces
via COM vtables.

WinUI 3 types (`Microsoft.UI.Xaml.*`) are **not** included in the `windows` crate.
They are generated at build time from the WinAppSDK `.winmd` metadata using
`windows-bindgen`. Run the setup script once to download the metadata:

```powershell
pwsh -File scripts/download-metadata.ps1
```

This language pack adds:

- **Bootstrap** — DDLM initialization via `MddBootstrapInitialize`
- **XAML Loader** — safe wrapper around `XamlReader::Load()`
- **Binder** — `FindName()` + `winui_view!` macro for struct field mapping
- **CLI** — `winuidev init / run / build`

## Cargo Features

```toml
[dependencies]
windows = { version = "0.58", features = [
    "Foundation",
    "implement",
]}

[build-dependencies]
windows-bindgen = "0.58"
```

WinUI 3 bindings are generated by `build.rs` from `.winmd` files in `metadata/`.

## Requirements

- Rust 1.70+ (stable)
- Windows 10/11 (x64)
- Windows App SDK 1.8
